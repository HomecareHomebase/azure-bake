# Test & Coverage Report (2026-01-22) - Final

This report captures the final test run and code coverage for the Azure Bake monorepo, along with all fixes and tests added to improve coverage without changing runtime behavior.

## Coverage commands

Run coverage tests from the repository root:

| Command | Description |
|---------|-------------|
| `npm run test:coverage` | Run tests with full coverage output (per-file stats + summary) |
| `npm run test:coverage:summary` | Run tests with summary-only coverage output (no per-file stats) |

Both commands run the full test suite across all packages. The difference is only in the coverage report output format.

**Configuration**: Coverage settings are defined in `.nycrc.json` at the repository root.

## Test tooling
- **mocha** – test runner
- **chai** – assertion library
- **sinon** – mocking, stubbing, and spying library
- **nyc** – code coverage instrumentation

## Plan review snapshot
The upgrade plan in `docs/upgrade.md` is intentionally conservative and test-first: it codifies current behavior through characterization tests before any dependency/tooling upgrades. The gating strategy is clear and safe (one major risk at a time, reversible steps, no behavior changes without tests). It aligns with CI constraints and the Docker runtime distribution model.
