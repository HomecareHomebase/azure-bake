# Test & Coverage Report (2026-01-21)

This report captures the latest full test run and code coverage for the Azure Bake monorepo, along with the fixes and tests added to improve coverage without changing runtime behavior.

## Plan review snapshot
The upgrade plan in `docs/upgrade.md` is intentionally conservative and test-first: it codifies current behavior through characterization tests before any dependency/tooling upgrades. The gating strategy is clear and safe (one major risk at a time, reversible steps, no behavior changes without tests). It aligns with CI constraints and the Docker runtime distribution model.

## Test run summary
- Full monorepo test run completed successfully via `nyc lerna run test`.
- Contract tests remain intentionally skipped where execution is unsafe (external services, dynamic installs).

## Fixes and tests applied to get green tests + coverage
- `tests/ingredient-contracts.test.ts`
  - Snapshot ARM params before plugins mutate them, preventing assertion failures when plugins remove or alter values.
  - Stubbed `@azure/arm-sql` to keep SQL DWH contract tests deterministic and avoid runtime `undefined` errors.
- `ingredient/ingredient-sql-dwh/package.json`
  - Switched to `ts-node/register/transpile-only` for the contract test runner to avoid duplicate type identity errors.
- `.nycrc.json`
  - Use `ts-node/register/transpile-only` so nyc can instrument code without cross-package type identity conflicts.
- `tsconfig.json` (new, repo root)
  - Minimal compiler options to enable JSON module imports and downlevel iteration during nyc runs.
- `arm-helper/test/arm-helper.test.ts`
  - Added unit tests for diagnostics configuration, alert parameter expansion, tag appending, and deep merge behavior.
- `ingredient/ingredient-property-service/test/*`
  - Added unit tests for configuration counts/validation, string utilities, and validation helpers.
- `core/test/utilities.test.ts`
  - Added unit tests for tag generation, base utility parsing, objToVariableMap, and BaseIngredient defaults.
- `core/test/bake-variable.test.ts`
  - Added tests for literal values, eval execution, and compile caching.
- `core/test/logger.test.ts`
  - Added log-level coverage for debug/info/warn/error behavior.
- `ingredient/ingredient-utils/test/core-utils.test.ts`
  - Expanded coverage for naming helpers, region selection, variable resolution, and conversions.
- `ingredient/ingredient-property-service/test/operations.test.ts`
  - Added tests covering OperationBase helpers and Property/Secret operations.
- `system/test/bake-loader.install.test.ts`
  - Updated to use symlinked ingredients to avoid npm installs during tests.
